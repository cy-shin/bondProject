## 5. 트러블 슈팅
<details>
<summary> DB 소실 문제</summary>

</br>

- 추가 기능을 지원하기 위해서 학원 DB의 서버가 업데이트되었는데, 업데이트 과정에서 문제가 발생하여 DB 내용이 전부 삭제되는 문제가 있었습니다.
- 이전에 저장해둔 SQL문과 샘플 데이터를 이용해 금방 복구할 수 있었습니다.
- 복구에 사용한 자료들이 최신 자료가 아니었기 때문에 DB 복구 후 자료들을 최신화해 백업해둔 후, 조원들 각자 개발환경에 별도로 DB를 생성해두어 DB 서버가 꺼졌을 때에도 프로젝트를 계속 진행할 수 있었습니다. 
- 백업의 중요성을 배웠고, 이후에는 DB 뿐 아니라 프로젝트도 백업하며 만일의 사태에 대비하게 되었습니다

</details>
